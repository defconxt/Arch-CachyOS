# VR Setup Guide (WiVRn + Meta Quest 3)

WiVRn wireless VR streaming on CachyOS with Meta Quest 3 and DCS World. NVIDIA RTX
5090, Wi-Fi 6. Covers automated install, dashboard settings, Quest 3 pairing,
OpenXR verification, and troubleshooting.

---

## TL;DR

1. Run `./scripts/07-vr-setup.sh` to install WiVRn, avahi, and xrizer
2. Pair Quest 3 via the WiVRn dashboard — see [Pairing Quest 3](#pairing-quest-3)
3. Launch DCS in VR: `~/.local/bin/launch-dcs-vr.sh -n`

Scripts: [`scripts/07-vr-setup.sh`](../scripts/07-vr-setup.sh) |
[`scripts/04-dcs-launch-vr.sh`](../scripts/04-dcs-launch-vr.sh)

---

## Prerequisites

- `docs/02-gaming.md` setup complete (Steam, GE-Proton installed)
- `docs/04-dcs-world.md` setup complete (DCS prefix configured, `04-dcs-setup.sh` run)
- NVIDIA driver 565 or newer:
  ```bash
  nvidia-smi | grep "Driver Version"
  ```
- Meta Quest 3 on the same Wi-Fi 6 (5 GHz) network as the PC
- `yay` (AUR helper) installed

---

## The VR Stack

```
DCS.exe (OpenVR game inside Proton/pressure-vessel)
    → xrizer (OpenVR → OpenXR bridge)
    → WiVRn (OpenXR runtime — host)
    → Wi-Fi 6
    → Meta Quest 3 (AV1 hardware decode)
```

**SteamVR is NOT used.** WiVRn is the OpenXR runtime. xrizer bridges DCS's legacy
OpenVR calls to the OpenXR API that WiVRn speaks.

## Install

```bash
./scripts/07-vr-setup.sh
```

**What the script does:**

| Step | What it does |
|------|-------------|
| `step_pacman_deps()` | Installs `avahi` and `nss-mdns` via pacman |
| `step_aur_packages()` | Installs `wivrn-server 26.2.3`, `wivrn-dashboard 26.2.3`, `xrizer-git` via yay |
| `step_avahi_mdns_conflict()` | Detects systemd-resolved mDNS conflict — **exits with instructions** if found |
| `step_enable_avahi()` | Enables and starts `avahi-daemon.service` |
| `step_enable_wivrn()` | Enables and starts `wivrn.service` (user service) |
| `step_firewall_rules()` | Opens ports 5353/udp and 9757 (nftables, session-only) |
| `step_verify_openxr()` | Verifies `active_runtime.json` points to WiVRn |
| `step_group_warning()` | Reminds you to log out so group membership takes effect |

> **Re-login required:** After the script completes, log out and back in. New group
> membership (`wivrn`, `render`) is not active until the next session.
>
> Verify after re-login:
> ```bash
> groups | grep -E 'wivrn|render'
> ```

### Persistent Firewall Rules

The script's nftables rules are session-only (cleared on reboot). For persistent
rules, add them to your system nftables config.

**nftables example** (scope to LAN interface for security):

```
table inet filter {
    chain input {
        # WiVRn — mDNS discovery and streaming
        iifname "enp*" udp dport 5353 accept comment "WiVRn mDNS"
        iifname "enp*" tcp dport 9757 accept comment "WiVRn stream"
        iifname "enp*" udp dport 9757 accept comment "WiVRn stream"
    }
}
```

**ufw equivalent:**

```bash
sudo ufw allow in on enp+ to any port 5353 proto udp comment "WiVRn mDNS"
sudo ufw allow in on enp+ to any port 9757 comment "WiVRn stream"
```

Replace `enp*` / `enp+` with your actual LAN interface name (`ip link` to check).

## WiVRn Dashboard Settings (RTX 5090 + Quest 3)

Configure these in the WiVRn dashboard (`wivrn-dashboard`) before launching DCS.
These are tested values for the RTX 5090 + Quest 3 combination.

| Setting | Value | Notes |
|---------|-------|-------|
| Encoder | nvenc | NVIDIA hardware encoder (RTX 5090) |
| Codec | AV1 | Quest 3 has hardware AV1 decode — best quality/bitrate ratio |
| Bitrate | 200 Mbps | Wi-Fi 6, 5 GHz — reduce to 150 if frame times degrade |
| Foveation | ON | Reduces peripheral resolution, increases center clarity |
| Resolution | 2064 × 2208 per eye | Quest 3 native — do not stack with DCS Pixel Density |

**AV1 bitrate note:** WiVRn issue #693 documents an AV1 10-bit bitrate regression
in some builds. If AV1 produces unexpectedly low quality at high bitrate settings,
verify the installed version:

```bash
pacman -Qi wivrn-server | grep Version
```

Check the WiVRn release notes for the version you have installed.

### Manual Config File

If you prefer to edit the config directly instead of using the dashboard:

**Location:** `~/.config/wivrn/config.json`

**Baseline:**

```json
{
  "encoders": [
    {
      "encoder": "nvenc",
      "codec": "av1",
      "width": 1.0,
      "height": 1.0,
      "bitrate": 200000000
    }
  ]
}
```

## Pairing Quest 3

These steps cannot be automated — they require GUI interaction and headset input.

1. Start the WiVRn dashboard on the PC:
   ```bash
   wivrn-dashboard
   ```
   (or launch from your app launcher)

2. Click **Start server** in the dashboard

3. On the Quest 3: open the **WiVRn app**
   - The app must be sideloaded from WiVRn GitHub releases — see version note below.
     Sideload via [SideQuest](https://sidequestvr.com/) or with
     `adb install wivrn.apk` (requires USB debugging enabled on the Quest — see
     Meta's developer documentation).
   - The app version must match the `wivrn-server` package version

4. Quest 3 discovers the PC via mDNS and displays a pairing PIN

5. Enter the PIN in the PC dashboard

6. The headset shows **"Connection ready"** — pairing is complete

> **Version match required:** The Quest 3 WiVRn APK and the `wivrn-server` package
> MUST be the same version. After `pacman -Syu`, check:
>
> ```bash
> pacman -Qi wivrn-server | grep Version
> ```
>
> If the version changed, re-sideload the matching APK from:
> https://github.com/WiVRn/WiVRn/releases

## Verification Checklist

Run these after install and re-login to confirm everything is working before
pairing the headset.

- [ ] WiVRn user service active:
  ```bash
  systemctl --user status wivrn.service
  ```
  Expected: `active (running)`

- [ ] avahi-daemon active:
  ```bash
  systemctl status avahi-daemon
  ```
  Expected: `active (running)`

- [ ] OpenXR runtime points to WiVRn:
  ```bash
  cat ~/.config/openxr/1/active_runtime.json
  # or: cat /etc/xdg/openxr/1/active_runtime.json
  ```
  Expected: path contains `wivrn`

- [ ] User is in required groups (after re-login):
  ```bash
  groups | grep -E 'wivrn|render'
  ```

- [ ] Firewall allows discovery ports (headset should find PC in WiVRn app):
  Ports `5353/udp` and `9757/tcp+udp` must be reachable from headset

- [ ] Quest 3 WiVRn app shows **"Connection ready"** after clicking Start in dashboard

## Troubleshooting

| Symptom | Likely Cause | Fix |
|---------|-------------|-----|
| Quest 3 can't find PC on network | avahi not running, or systemd-resolved mDNS conflict | `systemctl status avahi-daemon`; check `/etc/nsswitch.conf` for `mdns` entry |
| DCS launches in desktop (pancake) mode | WiVRn not active when DCS started, or wrong launch script | Ensure `wivrn.service` active; use `launch-dcs-vr.sh` not `launch-dcs.sh` |
| Headset connects but session disconnects immediately | Server/client version mismatch | Re-sideload APK matching `pacman -Qi wivrn-server` version |
| Heavy judder or NVENC at 100% | Encoder misconfiguration or bitrate too high | Set explicit codec/bitrate in `wivrn-dashboard` or `config.json`; try reducing bitrate to 150 Mbps |
| Permission denied on WiVRn socket | Group membership not active | Log out and back in; verify with `groups \| grep -E 'wivrn\|render'` |
| Headset shows pairing but disconnects on PIN entry | mDNS partially broken (resolved conflict) | Run `07-vr-setup.sh` again — it will detect the conflict and exit with remediation steps |
